# スマート購入プランナー 機能改修報告書

## 1. 変更内容まとめ

### ■ 変更前の仕様（課題）
- **検索の曖昧さ**: ユーザーが入力した文字列に対して「なんとなく」一致する商品を表示しており、「asdf」や「テスト」といった無意味な入力に対しても、全商品からランダムに、あるいは特定のカテゴリ（食料品等）が強制的に表示されてしまう問題がありました。
- **提案の画一性**: 予算設定に関わらず「最安」と「最新」の2パターンしか出せず、高予算を設定しても安価なものばかり提案されるなど、ユーザーのニーズ（高級志向など）に応えられていませんでした。

### ■ 変更後の仕様
キーワードからカテゴリを特定し、そのカテゴリ内で最適なセットを組み立てる「厳格なマッピング方式」へ移行しました。

1. **解析フロー**: `ユーザー入力（用途）` -> `マッピングテーブル（完全一致・部分一致）` -> `対応カテゴリ特定` -> `商品選定`
2. **planner_keyword_categories の役割**: ユーザーが入力する「やりたいこと（キーワード）」と、システム上の「商品カテゴリ」を紐付けるマスターデータとして機能します。これにより、プログラムを書き換えずに検索精度の調整が可能になりました。
3. **商品取得ロジックの厳格化**:
   - 2文字未満の入力や、無意味な文字列（asdf等）をブロックするバリデーションを追加。
   - マッピングテーブルに存在しないキーワードの場合は、中途半端な結果を出さず「該当なし」を返します。
   - 在庫なし、または削除フラグが立っている商品は完全に除外されます。
4. **3プラン生成ロジック**:
   - **コスパ重視プラン**: 各カテゴリで「最も安価な」商品を選択。
   - **バランスプラン**: 各カテゴリの価格帯で「中央値（中価格帯）」の商品を選択。
   - **ハイエンドプラン**: 予算上限ギリギリまで使い、各カテゴリで「最も高価な」商品を選択。予算オーバー時は順次グレードを下げる調整ロジックを搭載。

## 2. 使い方（ユーザー向け）

### ■ 基本的な使い方
1. **入力**: メインメニューの「プランナー」をクリックし、専用画面へ移動します。
2. **用途と予算**: 「ゲーム」「読書」「自炊」など、あなたの目的を自由に入力し、出せる予算の上限を選択して「提案を表示する」ボタンを押してください。
3. **結果**: 入力内容に基づいた3つの購入プランが提示されます。

### ■ 表示内容
結果画面では、プランごとに以下の情報が表示されます：
- **プラン名**: コスパ重視 / バランス / ハイエンド
- **構成商品詳細**: 各商品の「商品名」「価格」「カテゴリ」および商品詳細へのリンク。
- **合計金額**: セットで購入した場合の総額。
- **残予算**: あなたの予算に対して、あといくら余るか（節約できるか）。

### ■ 注意点
- **無効キーワード**: 「あ」などの短すぎる入力や、「asdf」などの意味のない入力、システムに登録されていない目的を入力した場合は、誤った提案を避けるため「該当商品なし」と表示されます。
- **DB依存**: この機能は最新の在庫状況と価格に基づいています。商品が1つも存在しないカテゴリが含まれる目的（キーワード）では、プランが生成されません。
