# Issue #27 スマート購入プランナー テスト環境構築ガイド

## 1. 機能概要

### 目的
「スマート購入プランナー」は、ユーザーが入力した「用途（キーワード）」と「予算」に合わせて、最適な商品の組み合わせを3つのプランで提案する機能です。これにより、ユーザーは自分の目的に合った商品を迷わず選ぶことができます。

### 仕組み
1. **キーワード解析**: ユーザーが入力した文章（例：「ゲームをしたい」）から、DBに登録されたキーワード（例：「ゲーム」）を抽出します。
2. **カテゴリ特定**: 抽出されたキーワードに紐づく商品のカテゴリ（例：「PC」「モニター」など）を特定します。
3. **商品選定**: 特定された各カテゴリの中から、以下の3つの基準で商品を1点ずつ選定し、セットを作成します。
   - **コスパ重視**: 各カテゴリで最も安い商品を選択。
   - **バランス**: 各カテゴリで中間の価格の商品を選択。
   - **ハイエンド**: 予算の範囲内で、各カテゴリから最も高価な商品を選択。

## 2. DB構成

本機能では以下の3つのテーブルを使用します。

| テーブル名 | 役割 |
| :--- | :--- |
| `planner_keyword_categories` | **新規追加**。ユーザーが入力する「用途キーワード」と、それに対応する「商品カテゴリ名」を紐付けるためのマッピングテーブルです。 |
| `categories` | 商品の分類を管理する既存テーブルです。プランナーはここで特定されたカテゴリIDを使用して商品を検索します。 |
| `items` | 個々の商品を管理する既存テーブルです。プランナーは「在庫あり(stock > 0)」かつ「削除フラグOFF(delete_flag = 0)」の商品のみを候補とします。 |

## 4. テストデータ (投入用SQL)

以下のSQLにより、プランナーに必要な環境をセットアップできます。（※詳細は `テスト用データベース.sql` を参照）

### カテゴリ・商品
```sql
-- カテゴリ追加
INSERT INTO categories VALUES(seq_categories.NEXTVAL, 'PC', 'デスクトップPCなど', DEFAULT, DEFAULT);
-- 商品追加 (例: PCカテゴリ)
INSERT INTO items VALUES(seq_items.NEXTVAL, 'ビジネスノートPC', 80000, '仕事用', 10, NULL, 3, 0, SYSDATE);
```

### キーワードマッピング
```sql
-- テーブル作成
CREATE TABLE planner_keyword_categories (
  id NUMBER(6) PRIMARY KEY,
  keyword VARCHAR2(100 CHAR) NOT NULL,
  category_name VARCHAR2(100 CHAR) NOT NULL
);
CREATE SEQUENCE seq_planner_keyword_categories NOCACHE;

-- データ登録例
INSERT INTO planner_keyword_categories VALUES(seq_planner_keyword_categories.NEXTVAL, 'ゲーム', 'PC');
INSERT INTO planner_keyword_categories VALUES(seq_planner_keyword_categories.NEXTVAL, 'ゲーム', 'モニター');
```

## 7. トラブルシューティング

| エラー/現象 | 原因 | 解決方法 |
| :--- | :--- | :--- |
| **ORA-00904** | テーブルまたはカラムが存在しない。 | `テスト用データベース.sql` を実行し、テーブルが作成されているか確認してください。 |
| **結果が出ない** | キーワードがDBに登録されていない。 | 「3. 利用可能キーワード一覧」にある正確な用語を入力してください。 |
| **商品が出ない** | 商品の在庫が0、または削除されている。 | `items` テーブルの `stock` を1以上に、`delete_flag` を0に設定してください。 |
| **プランが足りない** | 予算が不足している。 | 各カテゴリの最安商品の合計金額よりも予算を高く設定してください。 |
