# ガチャ・クーポン機能 詳細レポート

## 1. 機能概要
本機能は、ユーザーのエンゲージメント向上とリピート購入の促進を目的に、特定のイベント（ログイン・注文完了）時に抽選（ガチャ）を行い、当選者にクーポンを付与するシステムです。

### 主要な特徴
- **シームレスな体験**: モーダルウィンドウ形式を採用し、ページ遷移なしでガチャを実行可能。
- **動的な報酬**: 当選時にその場でユニークなクーポンコードを発行。
- **不正防止**: 実行回数の制限（レートリミット）および実行ログの記録。

## 2. システム構成・技術詳細

### 2.1 データモデル
- **Coupon (クーポン)**:
    - `code`: ユニークなクーポンコード（例: GACHA-XXXXXX）。
    - `discountType`: 割引種別（定額: amount / 定率: percent）。
    - `discountValue`: 割引値（500円 / 10%など）。
    - `validFrom / validUntil`: 有効期間（発行から30日間）。
    - `usageLimit`: 使用可能回数（デフォルト1回）。
    - `userId`: 発行先ユーザー。
- **GachaLog (ガチャ実行ログ)**:
    - 実行日時、ユーザー、イベント種別（login/order）、結果（win/lose）、IPアドレスを記録。

### 2.2 制御フロー
1.  **トリガー設定**:
    - `LoginController`: 一般ユーザーのログイン成功時に `canPlayGacha` フラグをセッションに設定。
    - `ClientOrderRegistController`: 注文完了時に `canPlayGacha` フラグを設定。
2.  **フロントエンド表示**:
    - `layout_5block.html`: セッションフラグを検知し、ガチャモーダルを自動表示。
3.  **抽選処理 (`GachaService`)**:
    - **レート制限**: 直近1分間に5回以上の試行がある場合は強制的に失敗。
    - **抽選ロジック**: 50%の確率で当選。
    - **報酬生成**: 「500円引き」または「10%OFF」のいずれかをランダムに選択し、DBに保存。
4.  **結果反映**:
    - `gacha.js` (AJAX): 演出後に結果を表示。当選時はコードを表示。

### 2.3 クーポン適用
- `ClientBasketController`: 買い物かご画面でコードを入力し適用。
- `PriceCalc`: 割引額を計算。注文確定時にクーポンの使用回数をデクリメント。

## 3. 不正・エラー対策
- **IPアドレス記録**: 同一ユーザーが複数アカウントを使い回すなどの不審な挙動を追跡可能。
- **セッション制御**: ガチャ実行後、またはモーダルを閉じた（キャンセル）時点で権限フラグを破棄し、二重実行を防止。

# 報告会での想定質問と回答 (Q&A)

### Q1. 当選確率（50%）はどこで管理されていますか？将来的な変更は容易ですか？
**A1.** 現在は `GachaService` クラス内の定数としてハードコードされています。将来的にキャンペーン等で頻繁に変更する予定がある場合は、設定ファイル（`application.properties`）や管理画面（データベース）から動的に取得する構成へ容易に拡張可能です。

### Q2. 複数のクーポンを同時に使用することは可能ですか？
**A2.** 現在の仕様では、1回の注文につき適用できるクーポンは1つまでとしています。これは過度な割引による利益圧迫を防ぐための安全策です。

### Q3. ログインするたびにガチャが引けるのですか？
**A3.** はい。ただし、短時間での繰り返しのログイン・ログアウトによる悪用を防ぐため、`GachaService` にて「1分間に5回まで」というレートリミットを設けています。

### Q4. スマートフォンでの表示は考慮されていますか？
**A4.** はい、CSSのメディアクエリおよびフレックスボックスを使用しており、レスポンスに対応しています。モーダルウィンドウも画面サイズに合わせて調整されます。

### Q5. クーポンの有効期限が切れた後の処理はどうなりますか？
**A5.** 買い物かごでの適用時に、システムが現在時刻とクーポンの `validUntil` を比較し、期限切れの場合はエラーメッセージを表示して適用を拒否します。
