# ガチャ・クーポン機能 詳細レポート

## 1. 機能概要
ユーザーのエンゲージメント向上を目的とした抽選システムです。ログインや注文完了をきっかけにガチャが起動し、当選するとクーポンが付与されます。

## 2. 特別設定 (本ブランチのみ)
報告会でのデモンストレーションを確実に行うため、**ガチャの当選確率を100%に設定**しています。
- 対象ファイル: `GachaService.java`
- 変更点: 当選判定ロジックを `roll < 100` に変更

## 報告会用 Q&A

### Q1. 当選確率は本番でも100%ですか？
いいえ、現在はデモンストレーション用の設定です。本番運用時にはビジネス要件に基づき、適切な確率（例: 20%〜30%）に変更します。

### Q2. 複数のクーポンを同時に使用することは可能ですか？
現在の仕様では、1回の注文につき適用できるクーポンは1つまでとしています。

### Q3. 不正対策はされていますか？
はい。同一ユーザーによる過度な実行を防ぐためのレート制限（1分間に5回まで）を実装しており、すべての実行履歴を `gacha_logs` テーブルに記録しています。
